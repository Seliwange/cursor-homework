• git config --global user.name blabla – ім’я, яке буде відображатися при створенні комітів • git config --global user.email blabal@bla – буде відображатися в інформації в створених комітах, ці дані вставляються в історію змін, тільки так можна дізнатись хто і коли зробив • git init – створює в локальній директорії сховану нову піддиректорію з ім’ям .git, в цій папці лежать всі необхідні файли для роботи системи git, вона починає індексувати всі файли в папці і починає індексувати історію, без неї не можна почати роботу • щоб перемістити локальний репозиторій на github треба git remote add origin [url – силка на репозиторій] • git status – відображає статус репозиторію • git add – команда підготовки файлів – індексація • git commit – операція, яка бере всі підготовлені зміни (можуть включати будь-яку кількість файлів) і відправляє їх в репозиторій як єдине ціле • git log – показує список всіх виконаних комітів, відсортованих по даті (зверху останні) • git log --oneline --decorate --graph –all – показує графік гілок і комітів • git checkout – переходить на потрібну гілку • git branch – створює нову гілку • git branch –d – видаляє гілку • git push – використовується для надсилання локальних змін на вказаний віддалений репозиторі; ця команда оновлює віддалений репозиторій, інтегруючи нові коміти • git pull – завантажує історію з віддаленого репозиторію і об’єднує її з локальною, pull = fetch + merge • git merge – злиття всіх гілок в одну • git clone – створює локальну копію віддалено репозиторію з github .gitignore - ігнорує файли, які не є частиною вихідного коду (службові файли, додані операційною системою, конфігураційні та тимчасові файли редакторів, логи, кеші, результати збірки проектів, залежності, результати виконання тестів) • git revert – створює новий коміт, на заміну від попереднього • git reset – повертає до потрібного коміту, видаляє коміт, який ще не відправлений у репозиторій • git rebase -i - спосіб комбінування змін з різних бранчів, бере кілька комітів , "копіює" їх, й кладе їх в інше місце, його використовують щоб створити зручну лінійну послідовність комітів • git cherry-pick – додає вибрані коміти до головної гілки • git stash – дозволяє ховати зміни в робочій директорії і відновити їх при необхідності, якщо потрібно робити термінові зміни в цих же файлах над якими працюємо • git stash pop – повернути останні зміни
